# 06 · UI 规格（Polite Cockpit）

> 本文件定义信息架构与行为约束。**视觉语言（调色板、字号阶、微交互）在实现阶段用 `ui-ux-pro-max` skill 现场决策**。定稿后回填本章节。

## 1. 定位

**一位 SRE 的驾驶舱。** 商业化克制、潮流克制、Polite（礼貌 —— 不打扰、不表演、不催促）。

参考气质：Linear 的清晰 · Vercel 的克制 · Warp 的工程感 · Notion 的对话流 —— 取其"单栏对话 + 固定命令输入"的范式，不取任何表情式装饰。

## 2. 信息架构（这是唯一的页面）

```
┌──────────────────────────────────────────────────────────────┐
│  ContextBar  (h≈40)                                          │
│  aws: default · gcp: sre-dev · k8s: prod · LLM: [ON/OFF]     │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Conversation (overflow-y, reverse? no — forward chrono)     │
│                                                              │
│   ─ Turn 1 ─────────────────────────────────────────         │
│     [user row]  /infra aws vm list --region us-east-1        │
│     [ResultCard]  table + 可选 LLM summary                    │
│                                                              │
│   ─ Turn 2 ─────────────────────────────────────────         │
│     [user row]  /cluster prod scale web --replicas 10 …      │
│     [PlanCard]  effect diff                                  │
│     [ApprovalCard]  [Reject] / [Approve]                     │
│     [RunCard]     streaming stdout                           │
│     [ResultCard]  "scaled deployment/web to 10 replicas"     │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│  CommandBar  (h≈64, sticky bottom)                           │
│  ┃1│ /infra aws vm list --region us-east-1   [↵ run]        │
│  ─ StatusLine ─────────────────────────────────────────      │
│  ● parsed · READ · infra.aws.vm.list                          │
└──────────────────────────────────────────────────────────────┘
```

**只有这一个页面**。没有 Sidebar。没有 tab。没有弹窗 Modal（除 danger 的二次确认输入条，但那是 in-place，不是弹窗）。

## 3. 组件清单

### 3.1 `<ContextBar>`（顶部）

一行，h40。左：profile 当前值（AWS / GCP / K8s 各一 chip，点击弹一个小下拉 list `/context` 返回的候选）。右：LLM 开关（switch）+ 一个 "clear" 按钮（清空当前对话流，不清审计）。

### 3.2 `<Conversation>`

正序时间线。每个 Turn 是一个 `<section>`。Turn 内从上往下：
1. `UserCommandRow`（用户敲了什么，带时间戳）
2. `PlanCard`（write 类才出）
3. `ApprovalCard`（write 类，审批未决时显示）
4. `RunCard`（执行中，stdout 流；执行完就折叠）
5. `ResultCard`（read 类直接出；write 类审批通过后出）
6. `ErrorCard`（任何阶段出错）

同一 Turn 只存在最多一个 ApprovalCard（被决策后 replace 为 RunCard / 或 Turn 终态为 Rejected）。

### 3.3 `<CommandBar>`（底部）

沿用 M1 的 CodeMirror 6 + StatusLine。细节：
- 输入时客户端做 token 着色（namespace / target / noun / verb / flag / value / ref）。
- 120ms debounce 后 `POST /parse`，结果回写 StatusLine。
- Enter：若 read → 直接 `POST /execute`；若 write → 也 `POST /execute`，但 API 返回 `awaiting_approval`，UI 立刻插入 PlanCard + ApprovalCard。
- 历史：↑/↓（Alt+↑/↓ 保留给编辑器）在空行或 Ctrl+R 查询最近 parsed 成功的命令。
- 不做多行。不做 stdin。不做 `|` 管道。严格单命令。

### 3.4 `<PlanCard>`

展示：skill id · risk label · before/after diff（monaco-diff 风格或简版 +/- 行）· rollback hint · 生效 profile / ns / region。**不可修改**，要改就另下命令。

### 3.5 `<ApprovalCard>`

两个按钮：`Reject` · `Approve`。`Reject` 点击后弹一行 inline `reason` 输入，回车提交。`Approve`：普通 skill 立即生效；`danger: true` 需先输入 `YES` 解锁。

### 3.6 `<RunCard>`

执行中：一个 stdout 区域（等宽、ANSI 支持、自动滚底）+ 一个小圆点 spinner。执行完：折叠为一行总结（耗时、exit code）。

### 3.7 `<ResultCard>`（关键）

四种渲染变体，由 skill `output.parse` + `output.kind` 决定：

1. **Table**（`kind: table`，`parse: json` + `columns` 声明）
   - 等宽列标题，斑马色可选（我想要 off）。
   - 支持列排序（点列头）、全局搜索（`/` 键）、导出 CSV / JSON。
   - 单元格 renderer：`state-badge`（pending / running / stopped / failed 配色）、`relative-time`（2 days ago）、`bytes`、`percent`、`link`。
   - 行数 ≥ 20 时自动分页 / 虚拟化。

2. **Structured**（单对象：`output: { kind: object }`）
   - Key-Value 表 + 右上角 "view raw JSON" 按钮折叠展开。

3. **Log**（`kind: log`）
   - xterm-like 等宽区域，支持选择复制、关键字高亮（ERROR / WARN）。

4. **Chart**（`kind: chart`）
   - 本 demo 只实现 2 种轻图：
     - `line` — 多系列折线（时间序列，如 `cluster.top.pod` 的 cpu/mem 走势）
     - `bar` — 水平条形（如资源排行榜）
   - 数据绑定：skill `output.chart = { type: line|bar, x, series: [...] }`。
   - 实现：SVG 手绘（100 行内），不引图表库。配色从 5 种 state 色取。悬浮显示精确值。
   - 图表下方同步显示一个紧凑数据表（可折叠），方便精确读数。

**LLM 摘要**（若 Context Bar LLM ON）在 Result 主区下方单独卡槽：
- 顶栏标识：`LLM·generated`（橙色小 chip）+ "hide" 按钮。
- 正文：`summary` 一句、`highlights` bullets、`findings`（带 level 配色：info/warn/error）、`suggested_commands`（灰色只读 chip，点一下只填 CommandBar 不运行）。
- 若后端判定 LLM output 与 raw 偏差大：顶栏红色 warning 条。

### 3.8 `<ErrorCard>`

What / Why / How 三段式。顶行红色 error code。中段自然语言描述（固定模板，不走 LLM；LLM 的解释可以以独立卡追加到这个 Turn）。底部可展开原始 stderr。

## 4. 视觉语言（最小合同，细节在 `ui-ux-pro-max` 阶段敲定）

**冻结的硬边界**（无论 skill 给什么方案都不能越过）：

| 规则 | 说明 |
| --- | --- |
| 深色 first，明亮 second，同等精度 | 两套主题等价实现 |
| Mono 用于命令 / 结果数据 / token；Sans 用于 UI chrome | 字体自选但不要 Inter / Arial |
| 无 emoji、无装饰性 gradient、无 glassmorphism | 徽章例外（✓ ✕） |
| Focus ring 2px，accent 颜色，键盘可达 | A11y 底线 |
| 语义色 ≠ 唯一信号 | 所有 state 同时带 glyph 或文案 |
| 一屏 1080p 下 ResultCard 可见 ≥ 20 行 | 信息密度底线 |
| LLM·generated 有固定视觉区隔 | 橙 chip + 边框颜色与 Result 不同 |
| write 命令全程有"这是写操作"的视觉暗示 | verb 异色 / PlanCard 左描边 / ApprovalCard 顶条 |

**实现阶段 `ui-ux-pro-max` 需要输出**：调色板（包含 5 种 state 色）、字号阶（5 档）、间距（4px 节律）、阴影规则、动效规则（应可用 prefers-reduced-motion 完全禁用）、徽章样式、表格样式。

## 5. 键盘映射

| 键 | 动作 |
| --- | --- |
| `Enter` | Submit CommandBar |
| `Esc` | 关闭补全 / 撤销 reject 输入 |
| `Ctrl+K` | Focus CommandBar |
| `Ctrl+L` | Clear conversation（不删审计） |
| `Ctrl+/` | Toggle LLM summary 开关 |
| `↑ / ↓` 空行 | 历史命令 |
| `Tab` 补全列表 | Accept |
| `A` / `R` 在 ApprovalCard 聚焦时 | Approve / Reject 触发（danger 不直接触发，仅聚焦到 YES 输入框） |

## 6. 实现顺序（UI）

1. **ContextBar**（最简单，先搭骨架）
2. **Conversation + UserCommandRow + ResultCard (table variant)**
3. **CommandBar（复用 M1 的 CodeMirror，删除旧 Home 的多区域）**
4. **PlanCard + ApprovalCard + RunCard（WS 流）**
5. **ErrorCard**
6. **LLM summary block（开关、标签、chip）**
7. **ui-ux-pro-max 终审：调色板、字号、微交互落地**

## 7. 明确删除（上一版做了但这一版不需要）

- Sidebar 导航（Home / Runs / Approvals / Skills / Audit / Ops）
- ExplainPane（合并入 ResultCard 的 LLM summary 块）
- CompletionPopover 独立组件 —— 改由 CodeMirror 内建 autocomplete 承担（锚点跟 cursor，体验更对）
- Home 的 EmptyOutput（改为对话流的初始态：一行 "Type a command to begin." + 3 个示例）

## 8. 可访问性

- 所有交互可键盘达到，焦点环清晰。
- ApprovalCard 的按钮有 `aria-describedby` 指向 PlanCard 摘要。
- `prefers-reduced-motion` 下所有动效立即关闭。
- 色盲：state 用 glyph / 文案双重标记，不单靠色。
