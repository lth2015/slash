# Slash — Design Context

> Frozen project context for design skills. Source: [docs/01-product-spec.md](docs/01-product-spec.md) & [docs/06-ui-ux-design.md](docs/06-ui-ux-design.md).

## Target audience

**Primary:** SRE / DevOps engineers and on-call engineers. They switch contexts every few minutes, are cold-read frequently (coming back to the tool mid-incident), and read fast. **Secondary:** platform engineers authoring Skills; SecOps / compliance reviewing audit trails.

They are opinionated power users. They dislike: AI-chat fluff, vague copy, being slowed by animations, losing keyboard control, unexplained errors, decorative cards, emoji.

## Use cases

1. Issue a single atomic command (`/infra aws vm list --region …`) and read results in a dense table.
2. Stage a write change (scale, rollback), review a Plan, request approval, execute, tail logs.
3. Inspect audit trail and Skill registry. Filter by user / command / time.
4. During incidents: glance at on-call / SLO / active alerts from Ops Dashboard.

## Brand personality / tone

- **Precise over friendly.** Technical terminology, no sugar-coating, no empty reassurance. Copy uses the "What / Why / How" error pattern.
- **Dense over spacious.** Information density approaches an IDE. Empty space is earned, not scattered.
- **Calm over noisy.** Motion is reserved for state transitions (parse, plan, approve, execute). No decorative transitions.
- **Keyboard-first.** Every flow must be completable via keyboard. Mouse is additive.
- **Instrumental.** The interface should feel like a cockpit panel or a Bloomberg terminal, not a marketing site.

Reference aesthetic: Linear × Vercel × Raycast, with the pragmatism of a Bloomberg terminal and the restraint of Dieter Rams.

## Hard constraints (from docs/06)

- **Dark-first**, light-mode at equal quality. No auto-switch on load.
- **No emoji** in product chrome; `✓` and `✕` OK as semantic glyphs.
- **No gradients**, no decorative drop shadows, no glassmorphism, no cards-in-cards.
- **No AI-chat input** in the command path. Only structured commands are executed.
- **Mono** for commands / logs / code. **Sans** for all UI chrome. Distinct font family pair, not Inter/system defaults.
- Must render at 1366×768 AND 1920×1080 without horizontal scroll. At 1080p the result table must show ≥20 rows above the fold.
- Color tokens are fixed (see `docs/06-ui-ux-design.md` §1.2 / `apps/web/tailwind.config.ts`). Do not add new named colors — compose with opacity or mix.

## What distinctive means here

The memorable thing is not a color or illustration — it is that **the command is the interface**. The editor is syntactically honest: what you see is what the parser gets. Every unresolved token has a concrete explanation nearby. The CommandBar feels like a legible piece of lab equipment, not a toy.
